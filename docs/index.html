Overview

Open OnDemand is a web portal for using parallel computer systems via a web browser. The applications provided by Fugaku Open OnDemand are divided into the following three groups.

  • Batch Jobs : Real world applications that run on Fugaku
  • Interactive Apps : Interactive applications that run on Fugaku and pre-post environment
  • Passenger Apps : Utilities that run on the server where Open OnDemand is installed

Gaussian, GaussView, MATLAB, and Vampir, which are commercial software, are also available, but their icons are displayed only for registered users in Fugaku portal.

Batch Jobs

Click the icon to display a web form for entering computational resources. Input items differ for each application. Only the important ones will be explained as following.

  • Elapsed Time : Maximum time used by the application
  • Mode : Execution mode for Fugaku. Please refer to HERE for detail
  • Version : Version number of the application

Clicking Launch bottom will submit the job to Fugaku.

Interactive Apps

Click the icon to display a web form for entering computational resources. Input items differ for each application. When fugaku or prepost is selected in Queue field, the input fields thereafter will change. Clicking Launch bottom will submit the job to the system selected in Queue field.

When a job is launched on a compute node, a link to the application is displayed. By clicking on the link, you can use the corresponding application from your web browser.

Passenger Apps

Active Jobs

You can view job information, and delete your jobs.

Home Directory

You can upload, download, delete, and edit files. The file size upload limit is 10GB.

File operations for cloud storage using rclone are also possible. You can set up rclone by starting Remote Desktop in Interactive Apps and running rclone or rclone-browser from the terminal. When you launch Home Directory afterwards, you can access the configured cloud storage. Please note that Fugaku Help Desk does not support rclone settings.

File operations for GakuNin RDM are also possible. When you launch GakuNin RDM in Passenger Apps, input following information.

  • MOUNT_POINT : mount point on Fugaku (e.g./home/rccs-aot/a00000/abc)
  • RDM_NODE_ID : A part of URL of your project in GakuNin RDM (xxx in https://rdm.nii.ac.jp/xxx/)
  • ROM_API_URL : https://api.rdm.nii.ac.jp/v2/
  • ROM_TOKEN : Parsonal access token in GakuNin RDM

When clicking mount botton, a project directory in GakuNin RDM is mount to Fugaku. When you want to unmount, please click unmount botton. Please note that Fugaku Help Desk does not support this application. Please visit to https://github.com/RCOSDP/CS-ood-rdmfs/.

Shell Access

You can perform operations using a command line interface on your web browser.

Note

  • The temporal directory for Open OnDemand is ${HOME}/ondemand, which is automatically created. There is no problem to delete it, but it also deletes user-installed extensions in Interactive Apps and history files.
  • Internet Explorer 11 is not supported
  • If you want to use copy and paste in Remote Desktop, please use the Chrome browser.
